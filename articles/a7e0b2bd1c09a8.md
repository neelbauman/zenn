# ライブラリにおける確実なシャットダウンとデータロスのジレンマ

## プログラムが止まる瞬間、データはどこへ行くのか。

Python のキャッシュライブラリ `beautyspot` は、背後で非同期に保存処理（I/O）を行っている。この「非同期」という仕組みは、メイン処理の高速化に貢献する一方で、プログラムが終了する瞬間に大きな課題を突きつけてくる。

もしユーザーが `exit()` を呼んだり、プログラムが最後の行まで到達したりしたとき、まだディスクに書き込まれていないキャッシュデータはどうなるのか。そのままプロセスが消滅すれば、データは虚空に消える。かといって、終了を延々と待ち続ければ、ユーザーは「プログラムがハングした」と感じるだろう。

本記事では、ライブラリの「終わり際」の振る舞いをどう設計し、信頼性とユーザー体験を両立させたかを整理する。

## 1. コンテキストマネージャによる「明示的な同期」

最も確実な方法は、ユーザーに「ここで終わる」という境界線を引いてもらうことだ。`beautyspot` では `with` ブロック（コンテキストマネージャ）の使用を推奨している。

```python
with spot:
    # 非同期にキャッシュが保存される
    heavy_task()
# ブロックを抜ける際、未完了の保存タスクが完了するのを自動的に待機（Flush）する
```

この設計の利点は、データのライフサイクルがコードの構造と一致することだ。スコープを抜ければ、すべての I/O が完了していることが論理的に保証される。ユーザーにとって「安心感」を最も提供しやすいインターフェースである。

## 2. `atexit` による「最後の防衛線」

しかし、すべてのユーザーが常に `with` ブロックを使ってくれるとは限らない。あるいは、予期せぬエラーでスコープを飛び出すこともあるだろう。

そこで、Python の `atexit` モジュールを使用して、プロセスの終了時に自動的にシャットダウン処理を走らせる「防衛線」を設けた。

```python
import atexit

def _cleanup():
    # プロセス終了時に未完了タスクを掃き出す
    spot.shutdown(wait=True)

atexit.register(_cleanup)
```

ただし、`atexit` には限界がある。強制終了（SIGKILL）や一部の致命的な C レベルのクラッシュでは実行されない。あくまで「正常な終了プロセス」における、ライブラリとしての誠実な後始末である（ADR-0051）。

## 3. 「待つ」か「捨てる」か：タイムアウトの設計

シャットダウン時に最も困難な判断は、「いつまで待つか」だ。

ネットワーク遅延やディスク I/O のスタックにより、保存処理が数分間終わらない可能性もある。ライブラリがそこで永遠にブロックし続けるのは、行儀が悪い。

`beautyspot` では、シャットダウンメソッドに `wait` と `timeout` パラメータを設けた。
*   **デフォルト**: 可能な限り待つが、ユーザーが明示的に `save_sync=False` を指定すれば、未完了タスクを破棄して即座に終了する。
*   **タイムアウト**: 一定時間待っても終わらない場合は、警告をログに残してプロセスを解放する。

「データの整合性」と「プロセスの応答性」。この相反する要求のどちらを優先するかを、最終的にユーザーが選択できるようにした。

## 4. リソースの解体順序（ADR-0051）

シャットダウンの内部実装において、リソースを解体する「順序」も重要だ。
1.  **新規タスクの遮断**: まず、新しい保存リクエストを拒否する状態にする。
2.  **イベントループの停止**: 保存を管理しているループを止め、キューに残ったタスクを掃き出す。
3.  **スレッドプールの終了**: 実際の I/O を行っているスレッドを閉じる。

この順序を一つでも誤ると、停止しようとしているスレッドに新しいタスクを投げようとして `RuntimeError` が発生したり、接続が切れた DB に書き込もうとしてクラッシュしたりする。v2.7.6 では、このシーケンスを ADR（アーキテクチャ決定記録）として厳密に定義し、クリーンな終了を実現した。

## まとめ

ライブラリの価値は、使っている最中だけでなく、使い終わる瞬間の「去り際」にも現れる。

*   **コンテキストマネージャ** で、明示的な同期の手段を提供する。
*   **`atexit`** で、暗黙的な後始末を保証する。
*   **解体順序** を厳密に制御し、終了時のエラーノイズを排除する。

「何事もなかったかのように、しかし確実に仕事を終えて立ち去る」。この終わりの美学が、ライブラリへの信頼を形作っている。
